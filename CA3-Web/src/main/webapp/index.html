The Backend

On the backend part of the project, we have created all the REST endpoints which are then used to fetch data from the client. The endpoints are as they follow: 1)/all - endpoint /places which fetches all the locations with their image, description, address and rating respectively. - here, people also create their user accounts by simply inputing information in the fields. All the data is then saved in the database through POST method. 2)/user - endpoint /createlocation where registered users can create a new location with a new image, description and address - endpoint /id which allows a user to search for a specific place given its id - endpoint /rate where user can rate a location Moreover, we created 4 entity classes respectively: Address, Place, Role and User. Those 4 entities represent our connection to the database. When we execute the server, our persistence.xml will hook up to the database and create all the data which we have preset in DeploymentConfiguration class.
